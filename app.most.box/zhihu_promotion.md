# 标题：我做了一个 Web3 笔记应用，想把数据的所有权真正还给你

在这个“云端”无处不在的时代，我们似乎习惯了把记忆交给大厂。
Notion 很强，但数据在别人的服务器上；Evernote 变慢了，会员费也涨了；网盘里的文件，可能因为一个违规词就变成“温馨提示”。

作为一名开发者，我常在想：**有没有一种方式，能让我的数据像影子一样，无论我走到哪里，只要我记得“我是谁”，数据就自然浮现？不需要担心服务器关停，不需要担心隐私泄露，甚至不需要备份助记词？**

于是，我开发了 **Most.Box**。

## 什么是 Most.Box？

Most.Box 是一个基于 Web3 技术的去中心化数字空间。你可以把它当作一个**永不丢失、绝对私密、抗审查的个人知识库**。

它不仅仅是一个 Markdown 笔记应用，更是一个基于 **IPFS** 和 **Crust Network** 的去中心化存储终端。

### 核心理念：如影随形

在 Web3 的世界里，通常我们要面对复杂的“助记词”（12个英文单词）。一旦丢了，资产全无；一旦被盗，神仙难救。这对普通用户太不友好了。

Most.Box 做了一个大胆的尝试——**抛弃助记词备份**。

在 Most.Box 中，你的身份（私钥）是由你的 **“用户名 + 密码”** 或 **“个性化签名”** 通过确定性算法（PBKDF2 + SHA256）实时计算生成的。
这意味着：

- **你不需要保存助记词文件**。
- **你的私钥从未在网络上传输**，它只在你的浏览器内存中短暂存在。
- **只要你脑海里的密码不忘，你的数字身份和数据就永远属于你**。

走到任何一台电脑前，打开网页，输入账号密码，你的所有数据（加密后的）就会从去中心化网络中被召回，在本地解密展示。这就是“如影随形”。

## 为什么它值得一试？

### 1. 真正的数据永久存储 (Permaweb)

Most.Box 深度集成了 **Crust Network**。这不仅仅是把文件上传到 IPFS 那么简单。
应用内置了与 Crust 链的交互逻辑。当你保存一份笔记或上传一张图片时：

1.  数据首先在本地被高强度加密（使用 NaCl 算法）。
2.  加密数据上传至 IPFS 网关。
3.  **关键一步**：应用会自动在 Crust 链上发起一笔“存储订单”（Storage Order）。
4.  全球的 Crust 节点会争抢这份订单，将你的数据分散存储在世界各地的服务器上。

只要 Crust 网络还在（目前已有数千个节点），你的数据就永远在线。

### 2. 极致的隐私保护

在 Most.Box 里，服务器（如果有的话）只是一个搬运工，它看不懂你的数据。
我们使用了 **X25519 (NaCl Box)** 进行端到端加密。

- 你的笔记标题、内容、上传的文件，在离开浏览器前就已经变成了乱码。
- 只有持有私钥（即知道密码）的你，才能解开这些数据。

### 3. 丝滑的写作体验

抛开 Web3 的硬核背景，Most.Box 本身也是一个好用的编辑器。
基于 **Milkdown** 构建，它提供了类似 Typora 的“所见即所得” Markdown 写作体验。

- 支持斜杠命令（/）快速插入。
- 代码高亮、数学公式、图表一应俱全。
- 极简的 UI 设计，专注于写作本身。

## 技术栈揭秘（给开发者朋友）

如果你也是技术爱好者，Most.Box 的架构或许你会感兴趣：

- **前端**: Next.js 16 (App Router) + React 19 + Mantine UI
- **状态管理**: Zustand
- **Web3 交互**: `@polkadot/api` (Substrate 链交互) + `ethers`
- **加密核心**: `tweetnacl` + `scure` 系列库
- **部署**: 纯静态导出 (SSG)，可以直接部署在 IPFS 上，实现 100% 去中心化前端。
